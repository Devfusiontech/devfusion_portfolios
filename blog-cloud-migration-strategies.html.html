Cloud

Pick the right path: rehost, replatform, or refactor

The best migration strategy balances time-to-value with future agility. Start with a well-governed landing zone, then move in waves with clear success metrics.

Landing Zone Checklist

  • Identity & access (AWS Organizations/Control Tower / Azure Management Groups).
  • Networking (hub-spoke, shared egress, DNS, private endpoints).
  • Security guardrails (SCPs/Policies), logging, backups.
  • Cost allocation tags and budgets from day zero.

Key Takeaways

  • Invest in a governed landing zone before moving workloads.
  • Migrate in waves; track KPIs like cost/unit and error budgets.
  • Plan cutover with replication and blue-green techniques.

FAQs

Which path should I pick? Rehost for speed, replatform for quick wins, refactor for long-term agility.

How do I control cost? Use budgets, tagging, rightsizing, and autoscaling; review after each wave.
